## Сквозной сценарий технолога (5 минут)

### Шаг 1: Создать деталь (30 сек)
1. **Меню** → «Новая деталь»
2. Заполните 10 полей:
   - **Обозначение** (обязательно, уникальное) — например, `ЛМША.305001.020`
   - **Название** (обязательно) — например, `Кронштейн`
   - **Уровень**: деталь / сборка / изделие / покупное / полуфабрикат
   - **Масса (кг)**, **Чертёж №**, **REF 1С**, **Тип** — опционально
3. **«Сохранить»** → переход на карточку детали

### Шаг 2: Сгенерировать ТК (24 сек — ИИ думает)
1. На карточке детали → **«Сгенерировать ТК»**
2. Откроется форма → **«Запустить генерацию»**
3. Подождите ~24 сек (ИИ создаёт операции)
4. Появятся 4-8 операций со светофором (зелёный = есть доказательство, жёлтый = предположение)

### Шаг 3: Поправить операции (1-2 мин)
1. Кликните на **название операции** или **время** → inline-edit
2. **Enter** = сохранить, **Escape** = отменить
3. Если время «жёлтое» — кликните **«Подтвердить норму»** → подтверждаете вручную
4. Если операция «красная» — скорее всего нужен другой workshop/equipment

### Шаг 4: Экспортировать в 1С (10 сек)
1. **«Экспорт в 1С»** → XML файл
2. Скачайте → загрузите в 1С:ERP

## Извещения об изменениях

### Создать извещение
1. **Меню** → «Извещения» → **«Новое извещение»**
2. Заполните: **Номер**, **Дата**, **Основание**, **Причина**, **Описание**, **Обозначение детали**
3. **«Сохранить»** → извещение создано

### Workflow извещения
1. **«AI diff»** — ИИ сравнивает старую и новую версии
2. 3 решения:
   - **«Принять AI»** — применить изменения ИИ
   - **«Ручная проверка»** — отметить для ручного разбора
   - **«Отклонить»** — отклонить изменения
3. После решения извещение закрывается

## База знаний (эталоны)

- **Меню** → «База знаний» → эталоны утверждённых ТК
- Используются для RAG (поиск похожих) при генерации новых ТК
- Гл. технолог может утвердить ТК → она попадает в эталоны

## Горячие клавиши и UX

- **Кнопка-ссылка** (не popup) — все действия через redirect
- **Inline-edit** в операциях — клик → input → Enter
- **Авто-сохранение** при approve (атомарная транзакция)
- **Светофор** показывает доказательство нормы: зелёный = факт, жёлтый = предположение

## Часто задаваемые вопросы

### Почему операция «жёлтая» (предположение AI)?
Нет аналога в базе знаний или аналог не полностью совпадает. Решение:
1. Подтвердить вручную (нажать «Подтвердить норму»)
2. Или изменить норму через inline-edit

### Что делать, если генерация «зависла»?
Подождите 60 сек — таймаут LLM. Если не помогло — обновите страницу, попробуйте снова.

### Где смотреть audit-trail операций?
В карточке детали → вкладка «История» (история изменений).
Полный audit в БД: таблица `history`.

### Что делать, если не работает кнопка?
1. Проверьте что есть права (роль)
2. Проверьте подключение к интернету
3. Обратитесь к администратору
