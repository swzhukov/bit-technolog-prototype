# PENALTIES — шкала штрафов за провалы

> **Цель:** сделать ошибки дорогими, повторение — невыносимым.
> **Применяется:** каждая итерация (Sprint, цикл аудита, fix).
> **Начисляет:** Mavis (сам, ретроспективно) → проверяет Сергей.
> **Порог:** **100 баллов = наказание** (см. ниже).

---

## ШКАЛА (4 tier)

### TIER 1 — ТЯЖЁЛЫЕ (40-50 баллов)

| Баллы | Нарушение |
|-------|-----------|
| **50** | Проигнорировал явное указание Сергея |
| **50** | Изменил/удалил prod данные без backup |
| **40** | Повторная ошибка, которая УЖЕ была и должна была быть в `memory` или `MISTAKES.md` |
| **40** | Сделал без спроса то, что НЕ было в плане спринта |
| **40** | Скрыл/не сообщил о найденном баге до того, как Сергей его нашёл сам |

### TIER 2 — СРЕДНИЕ (20-30 баллов)

| Баллы | Нарушение |
|-------|-----------|
| **30** | Повторный паттерн ошибки (тот же класс проблемы 3-й раз за спринт) |
| **30** | Сделал изменение в коде → сломал существующее (без `git diff`/`py_compile`/теста) |
| **25** | Исправлял что-то и не проанализировал влияние на остальное (CARTE.md, MISTAKES.md) |
| **25** | Не проверил schema/сигнатуру API перед деплоем (нашёл на prod) |
| **20** | Сделал commit без verify (smoke/health) после deploy |
| **20** | Забыл добавить в `memory_append` урок |

### TIER 3 — ЛЁГКИЕ (5-15 баллов)

| Баллы | Нарушение |
|-------|-----------|
| **15** | Сделал change без `git diff` после правки |
| **15** | Использовал test-data в prod (RBAC-, TEST-) |
| **10** | Не обновил `CARTE.md` после изменения endpoint/template |
| **10** | Не проверил syntax через `py_compile` перед commit |
| **10** | Оставил TODO/FIXME/XXX в коде |
| **5**  | Забыл backup перед destructive op (cleanup, migration) |

### TIER 4 — БОНУСЫ (отрицательные)

| Баллы | За что |
|-------|--------|
| **−10** | Нашёл баг сам, до того как Сергей его увидел |
| **−5**  | Добавил в `MISTAKES.md` проактивно (не по факту) |
| **−5**  | Сделал `worktree-management` правильно с первого раза |

---

## НАКАЗАНИЯ при 100+ баллов за спринт

| Сумма | Наказание |
|-------|-----------|
| **100-149** | Warning. Сергей выбирает: откатить последний коммит, или сделать пересмотр. |
| **150-199** | Sprint pause: только bug-fixes, новые фичи запрещены до разбора. |
| **200+**   | Hard reset: откат к HEAD на начало спринта, переделать с нуля. |
