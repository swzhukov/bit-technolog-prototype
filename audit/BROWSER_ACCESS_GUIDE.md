## Как зайти в браузере (Chromium / Yandex Browser / Edge)

### Способ 1: прямой URL
В адресной строке ЯВНО набрать с `https://`:
```
https://217.114.7.5:8081/
```
**НЕ** `217.114.7.5:8081` (это будет http, белый экран).

### Способ 2: через HTTP redirect
```
http://217.114.7.5:8082/
```
Это перенаправит на `https://...:8081/`.

### Что увидите после ввода URL:
1. **Большой красный экран "Подключение не защищено"** или "Сертификат недействителен"
2. Кнопка **"Дополнительно"** / **"Advanced"** — кликнуть
3. Появляется ссылка **"Перейти на сайт 217.114.7.5:8081 (небезопасно)"** — кликнуть
4. **Сайт откроется** → login form → "Вход — БИТ.Технолог"
5. Ввести `techadmin` / `demo` → дашборд

### Если ВООБЩЕ не открывается (даже ошибки нет)
Возможные причины:
- **Корпоративный VPN/прокси** — отключить
- **Расширение блокирует** — режим инкогнито
- **Браузер в режиме "Организация"** (Яндекс.Браузер) — отключить
- **Kaspersky / Dr.Web** с HTTPS-инспекцией — отключить

### Если белый экран
1. Открой **F12** (или **Ctrl+Shift+I**)
2. Должна открыться панель DevTools внизу
3. Перейди на вкладку **Network**
4. Перезагрузи страницу (F5)
5. Сделай скриншот — увидим, какие запросы, какие статусы

### Если F12 не работает (политика организации)
1. Введи `chrome://policy` в адресной строке
2. Посмотри, есть ли запрет на DevTools
3. **Установи Firefox Portable** с флешки: https://portableapps.com/apps/internet/firefox_portable
4. Открой в Firefox — там F12 работает

### Последнее средство: другое устройство
**Зайди с телефона через мобильный интернет** (отключи Wi-Fi):
- `https://217.114.7.5:8081/`
- Если на телефоне ОТКРЫВАЕТСЯ, а на рабочем компе нет → **корпоративный firewall/прокси** блокирует 8081/8082
- Решение: настроить проброс портов через корп VPN, или поднять на 80/443 (нужен домен + Let's Encrypt)
