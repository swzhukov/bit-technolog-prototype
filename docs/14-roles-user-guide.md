# Руководство пользователя по 7 ролям

> **Версия:** v0.4.12 (2026-07-20)
> **Для кого:** все пользователи БИТ.Технолог
> **Главный принцип:** система **визуально подсказывает**, что вы можете делать в текущей
> роли. Если кнопки нет — значит, для вашей роли это действие недоступно.

---

## Как войти в систему

1. Откройте в браузере `http://ваш-сервер:8081/` (например, `http://localhost:8081/` для локальной разработки)
2. Если у вас есть логин/пароль — введите их (на пилоте не используется, см. ниже)
3. Если вы на демо или пилоте — **выберите роль** в правом верхнем углу

### На пилоте (для Техинкома) — без пароля

В пилотном режиме **пароль не используется** — это on-premise система внутри
завода, пользователи переключают роль через UI (под кнопкой "Переключить роль"
в правом верхнем углу). В будущем добавим LDAP/AD-аутентификацию.

### 3 быстрые кнопки (для показа клиенту/руководству)

На главной странице есть синяя плашка **"🎬 Показ клиенту — быстрая смена роли"**
с тремя большими кнопками:

- **👨‍🔧 Технолог** — создаёт ТК
- **👑 Гл. технолог** — утверждает ТК
- **🛡 Админ** — управляет системой

Нажмите кнопку — страница перезагрузится, и **цветной badge в header покажет вашу роль**.

### Селектор ролей (для всех 7 ролей)

В header справа — выпадающий список со всеми 7 ролями:

- 👨‍🔧 Технолог
- 👑 Гл. технолог
- 📏 Нормировщик
- 📐 Конструктор
- 🏭 Нач. цеха
- 🔍 ОТК (контролёр)
- 🛡 Админ

### Текущая роль

В header справа от селектора — **цветной badge с иконкой и названием роли**:

- Синий = Технолог
- Фиолетовый = Гл. технолог
- Красный = Админ
- Голубой = Нормировщик
- Оранжевый = Нач. цеха
- Зелёный = Конструктор
- Розовый = ОТК

**Всегда видно, в какой роли вы сейчас работаете.**

## 1. Технолог (👨‍🔧) — основная роль

**Главный пользователь системы.** Создаёт техкарты.

### Что вы можете

✅ Создавать новые детали (`Детали → ＋ Деталь`)
✅ Запускать AI-генерацию ТК (3-step UX или кнопка "Сгенерировать")
✅ Редактировать операции (название, оборудование, время, материалы)
✅ Добавлять, удалять, восстанавливать операции
✅ Просматривать версии (draft v1, v2, v3) и сравнивать их (diff)
✅ Задавать правила для AI («Учитывать в AI»)
✅ Утверждать ТК (если это ваша деталь и она в статусе "Проект ТК")
✅ Возвращать в работу (reopen) утверждённую ТК
✅ Видеть все 25 seed-деталей Техинкома
✅ Импортировать детали (JSON, Excel, PDF — через `/api/import/tk`)
✅ Экспортировать в PDF, Excel, 1С:CSV
✅ Печатать с QR-кодом

### Что вы НЕ можете

❌ Видеть админ-панель (`/admin` — 403)
❌ Видеть настройки системы (`/admin/settings` — 403)
❌ Смотреть LLM-логи всех пользователей
❌ Управлять другими пользователями

### Типичный workflow (30 минут на деталь)

1. **Главная** → `＋ Деталь` → заполнить обозначение, наименование, шасси, материал, размер, массу, поверхностную обработку → `💾 Сохранить`
2. **Откроется карточка детали** в статусе "🔴 Новый"
3. **AI-помощник** (открыт по умолчанию):
   - **Шаг 1:** нажмите `🤔 Уточнить` → AI задаст 3-5 вопросов → ответьте на них
   - **Шаг 2:** нажмите `⚡ Draft` → AI создаст короткий маршрут (3 операции, ~1₽)
   - **Шаг 3:** нажмите `✨ Полная ТК` → AI создаст полный маршрут (~3₽)
4. **Просмотрите** операции. Кликните на название — отредактируйте. Скажите AI что-то в комментарии — пересоздаст.
5. **Проверьте экономику** во вкладке `💰 Экономика` — ставка, материал, накладные
6. **Утвердите:** галки в чеклисте (4 пункта) → `✅ Утвердить`
7. **Экспортируйте:** `📄 PDF` (для архива) или `📊 Excel` (для 1С) или `/api/export/onec-csv`

### Полезные фичи

- **`localStorage` answers** — ваши ответы на вопросы AI сохраняются в браузере, не теряются при обновлении страницы
- **Self-timer** — время работы с карточкой считается автоматически (для метрик пилота)
- **Pre-approve checklist** — 4 галочки, которые не дают утвердить ТК, пока не проверено всё
- **Soft-delete** — удалённые операции попадают в "Удалённые" секцию, можно восстановить

## 2. Гл. технолог (👑) — утверждающий

**Контролирует качество ТК и подписывает.** Второй по важности пользователь.

### Что вы можете

✅ **Всё, что технолог** (создавать, редактировать, генерировать)
✅ **Утверждать как гл. технолог** — после того как технолог утвердил проект
✅ **Возвращать в работу** утверждённые ТК
✅ **Видеть все детали** (свои + чужие)
✅ **Видеть** историю изменений (кто и когда менял)
✅ **Видеть** отчёты пилота (если знаете URL)

### Что вы НЕ можете

❌ Видеть админ-панель
❌ Менять глобальные настройки

### Типичный workflow

1. **Главная** → список деталей → фильтр `🟡 Проект ТК` (те, что технологи уже утвердили)
2. **Откройте деталь** → просмотрите маршрут, экономику, замечания
3. Если есть сомнения — **`Вернуть в работу`** → технолог получит уведомление
4. Если всё хорошо — **`Утвердить как гл. технолог`** → ТК уходит в архив, редактирование ограничено
5. **Экспортируйте** для архива: `📄 PDF` с QR-кодом

### Совет

Используйте **фильтр по статусу** в URL: `/?status=draft` покажет все проекты,
которые ждут вашего утверждения.

## 3. Нормировщик (📏) — контроль норм

**Проверяет, что трудоёмкость и ставки соответствуют нормативам.**

### Что вы можете

✅ **Просматривать** все детали и ТК (только чтение)
✅ **Видеть** экономику детально (ставки, материалы, накладные)
✅ **Сравнивать** эталонные бенчмарки (вкладка `📊 Бенчмарки`)
✅ **Смотреть** отчёты пилота

### Что вы НЕ можете

❌ Создавать детали
❌ Генерировать ТК через AI (кнопки нет, API → 403)
❌ Редактировать операции
❌ Утверждать
❌ Видеть админ-панель

### Типичный workflow

1. **Главная** → открываете интересующую деталь
2. **Вкладка `💰 Экономика`** — проверяете ставки (цех 1, цех 2, цех 3), стоимость материала, накладные
3. **Сравниваете** с эталонными нормативами в `📊 Бенчмарки`
4. Если расхождение > 20% — **пишете в Telegram** технологу: «Проверь ставку на операции 015»
5. Никаких кнопок не нажимаете — только смотрите

### Полезный URL

`/benchmarks` — таблица эталонных трудоёмкостей по типам операций

## 4. Конструктор (📐) — поставщик деталей

**Создаёт детали для технолога** (но пока не может генерировать ТК).

### Что вы можете

✅ **Создавать** детали
✅ **Заполнять** обозначение, наименование, чертёж, размеры
✅ **Просматривать** ТК (когда технолог их сделает)

### Что вы НЕ можете

❌ Генерировать ТК (кнопок нет, API → 403)
❌ Редактировать операции
❌ Утверждать

### Типичный workflow

1. **Главная** → `＋ Деталь` → заполнить все поля по чертежу → `💾 Сохранить`
2. **Сообщить технологу** (лично или через Telegram): «Деталь `АЦ-ХХХ-YYY` готова»
3. Технолог сам откроет, сгенерирует ТК, утвердит

## 5. Нач. цеха (🏭) — контроль производства

**Контролирует, что ТК реализуема в цеху.**

### Что вы можете

✅ **Просматривать** все утверждённые ТК
✅ **Просматривать** версии и историю
✅ **Сравнивать** с эталонными бенчмарками
✅ **Утверждать как начальник цеха** (после гл.технолога, для финального ОТК)

### Что вы НЕ можете

❌ Создавать детали
❌ Генерировать ТК
❌ Редактировать операции
❌ Видеть админ-панель

### Типичный workflow

1. **Главная** → фильтр `🟢 Утверждён` (финальные ТК)
2. **Откройте** деталь → проверьте, что:
   - Оборудование есть в вашем цеху
   - Время реалистично
   - Материалы доступны
3. **Утвердите как начальник цеха** или верните в работу с комментарием

## 6. ОТК / Контролёр (🔍) — контроль качества

**Проверяет, что ТК соответствует требованиям качества.**

### Что вы можете

✅ **Просматривать** все детали и ТК
✅ **Видеть** контрольные точки (в операциях)
✅ **Сравнивать** с эталонными бенчмарками
✅ **Смотреть** отчёты

### Что вы НЕ можете

❌ Создавать детали
❌ Генерировать ТК
❌ Редактировать
❌ Утверждать (ваша роль только для просмотра и контроля)

### Типичный workflow

1. **Главная** → открываете деталь
2. Смотрите на **контрольные точки** в каждой операции (поле `control_points` в JSON)
3. Если что-то не так — **сообщаете** гл.технологу

## 7. Админ (🛡) — управление системой

**Единственная роль с доступом к админ-панели.** Управляет настройками, мониторит систему, смотрит логи.

### Что вы можете

✅ **Всё, что технолог + гл.технолог**
✅ **Доступ к `/admin`** — 8 страниц:
  - `/admin` — главная админки (метрики, статус)
  - `/admin/users` — управление пользователями
  - `/admin/login-log` — кто когда входил
  - `/admin/llm-calls` — все вызовы LLM, цена, ошибки
  - `/admin/settings` — настройки (YandexGPT API key, Telegram bot, SMTP)
  - `/admin/errors` — последние 50 ошибок сервера (V8-18)
  - `/admin/system` — состояние системы (БД, RAG, диск)
  - `/admin/backup` — ручной бэкап
✅ **Резервное копирование** (ежедневно 3:00, retention 30 дней)
✅ **Расшифровка бэкапов** (если включён gpg)
✅ **Retention policy** — очистка audit_logins 180д, llm_calls 90д, history 365д
✅ **Health-check** — мониторинг `/health` каждые 5 минут + Telegram алерт

### Что вы НЕ можете

❌ Редактировать ТК других пользователей (кроме как admin + override)

### Типичные задачи

#### Добавить YandexGPT API key
1. Переключитесь в роль **Админ**
2. **Настройки** (`/admin/settings`) → группа **LLM (YandexGPT)**
3. Введите API key → `💾 Сохранить`
4. Система автоматически переключится с demo-режима на реальный LLM

#### Посмотреть, что LLM тратит
1. **LLM-вызовы** (`/admin/llm-calls`) — таблица всех вызовов
2. Столбцы: время, деталь, модель, токены, стоимость, ошибка
3. Фильтр по дате, по детали, по статусу

#### Скачать бэкап
1. **Бэкап** (`/admin/backup`) → `💾 Скачать .db`
2. Или через CLI на сервере: `cd /opt/beget/bit-technolog && bash backup.sh`

#### Посмотреть ошибки
1. **Ошибки** (`/admin/errors`) — последние 50 ошибок с stacktrace
2. Каждая ошибка имеет 8-символьный debugging ID — ищите в логах

#### Обновить систему
1. SSH на сервер: `ssh user@server`
2. `cd /opt/beget/bit-technolog && git pull && bash deploy.sh`
3. Скрипт: обновит код, перезапустит systemd, прогон тестов, очистит кэш

Полный гайд — см. [`12-admin-guide.md`](12-admin-guide.md).

## Частые сценарии

### "Хочу посмотреть, как работает утверждение ТК"

1. Главная → `＋ Деталь` → создайте любую
2. Откройте её → AI-помощник → `⚡ Draft`
3. Дождитесь (5-10 сек в demo, 30-60 сек в LLM)
4. `✅ Утвердить` → галки в чеклисте → `✅ Да, утвердить`
5. Теперь переключитесь в роль **Гл. технолог** (через селектор)
6. Откройте ту же деталь → появится кнопка `Утвердить как гл. технолог`
7. Нажмите → статус станет `🟢 Утверждён`

### "Хочу импортировать список деталей из Excel"

1. Подготовьте файл: `id, designation, name, model, chassis, material, size_mm, mass_kg, surface_treatment`
2. Главная → `＋ Деталь` → `Импорт` (в будущем) или через API:
   ```bash
   curl -X POST http://server:8081/api/import/tk \
     -F "file=@details.csv"
   ```
3. Система создаст детали, проиндексирует в RAG

### "Хочу посмотреть отчёт пилота"

Главная → `📈 Отчёт` (`/pilot/report`) — сводка за неделю/месяц:
- Принято/отклонено ТК
- Стоимость LLM за период
- Acceptance rate (% принятых draft'ов)
- Время на деталь (медиана)
