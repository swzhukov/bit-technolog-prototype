# ADR-0005: 7 специализированных промтов вместо одного общего

**Дата:** 2026-05-10
**Статус:** Accepted
**Контекст:** Генерация ТК включает операции разного типа: сварка, электромонтаж, гидравлика, окраска, механообработка, общая сборка.

## Решение

7 разных system prompts в `prompts.py`:
- `WELDING_PROMPT` — сварка (MIG/MAG/TIG, режимы по толщине, проволока Св-08Г2С, газ М21)
- `ELECTRICAL_PROMPT` — электромонтаж (провода ПВА/ПГВА, клеммы WAGO, спецсигналы)
- `HYDRAULIC_PROMPT` — гидравлика (РВД, шланги, фитинги)
- `PAINT_PROMPT` — окраска (грунты ГФ-021, эмали ПФ-115, порошковая)
- `TECH_CARD_PROMPT` — общая ТК (механообработка, сборка, упаковка)
- `CLARIFICATION_PROMPT` — задаёт уточняющие вопросы
- `REFINE_PROMPT` — дорабатывает draft до полной ТК

## Обоснование

**Плюсы:**
- LLM получает специализированный контекст (для сварки — таблицу режимов по толщине, для электрики — марки проводов)
- Лучшее качество (промт фокусируется на одной области)
- Легче отлаживать (если сварка кривая — фиксим WELDING_PROMPT, не ломаем всё)

**Минусы:**
- 7× больше текста для поддержки
- 7× больше мест где LLM может галлюцинировать
- Нужно решить какой промт когда использовать (логика в `app.py:generate`)

## Как выбирается промт

Сейчас выбор по типу операции (первая операция в маршруте). В будущем — по типу детали (деталь из сварки → WELDING_PROMPT).
