# ADR-0002: YandexGPT (on-premise через cloud) для генерации ТК

**Дата:** 2026-04-20
**Статус:** Accepted
**Контекст:** Нужен LLM для генерации черновиков техкарт. Данные НЕ должны покидать периметр завода (ГОЗ).

## Решение

YandexGPT через OpenAI-совместимый SDK с сервера Техинкома (не передаём данные на внешние сервера).

## Обоснование

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **YandexGPT** ✅ | Yandex — российский провайдер, сертификация, OpenAI API совместим, 200₽/день | Vendor lock-in |
| Локальный LLM (ollama + Llama 3) | Полный on-prem | Нужна GPU-нода, 100GB+ RAM, сложно поддерживать |
| OpenAI/Anthropic | Лучшее качество | Данные уходят за рубеж — нарушение ГОЗ |
| GigaChat (Сбер) | Российский | Менее стабильный, нет OpenAI совместимости |

## Последствия

**Плюсы:**
- Данные не покидают РФ
- Дёшево (200₽/день при 50 ТК/день)
- OpenAI-совместимый API → легко мигрировать

**Минусы:**
- Vendor lock-in (но OpenAI API мигрирует за 1 день)
- Качество может быть ниже GPT-4
- Нужна валидация ключа

## Когда пересматривать

- Если понадобится локальный LLM (нужна GPU)
- Если появится сертифицированный open-source LLM лучшего качества
- Если Техинком переедет в другой cloud
