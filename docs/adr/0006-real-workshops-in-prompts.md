# ADR-0006: Реальные цеха и операции Техинкома в LLM промте

**Дата:** 2026-07-20 (M28)
**Статус:** Accepted
**Контекст:** LLM генерирует ТК с абстрактными операциями ("Сварка", "Гибка") и абстрактным оборудованием. Технолог потом правит. Хотим — LLM предлагает РЕАЛЬНЫЕ операции на РЕАЛЬНОМ оборудовании.

## Решение

Файл `workshops_tehinkom.py` с константой `TECHINKOM_WORKSHOPS_CONTEXT` (2190 символов):
- 4 production (УОМ, Мелкосерийное, ПВТ, ПКТ)
- 7 workshop'ов (Уч-к лазерной резки, Уч-к гибки, Сварочно-сборочный уч-к КТ, ...)
- 36 операций (Резка на лазерном станке, Сборка и прихватка цистерны 53-ТВ.05.00.00, ...)

Подставляется в `$workshops_context` в `TECH_CARD_PROMPT` и `REFINE_PROMPT`.

## Обоснование

**До M28:** LLM говорил "сварка на сварочном аппарате" — технолог правил "Кедр-300".

**После M28:** LLM говорит "Сварка узлов растяжек на уч-ке изготовления растяжек (Мелкосерийное производство)" — сразу конкретика.

**Плюсы:**
- Меньше правок технолога (точность растёт)
- LLM не выдумывает операции которых нет
- Понятная структура маршрута

**Минусы:**
- При изменении цехов — нужно обновлять файл
- Привязка к одному заводу (нельзя reuse для другого без правки)

## Источник данных

DOCX "Участки производства Техинком-Центр" (июль 2025). Распарсен в `/workspace/attachments/parse_workshops.py`.
