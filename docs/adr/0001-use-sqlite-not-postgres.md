# ADR-0001: SQLite вместо PostgreSQL

**Дата:** 2026-04-15
**Статус:** Accepted
**Контекст:** БИТ.Технолог — on-premise продукт для ООО «ПК Техинком-Центр» (ГОЗ, оборонка). Один завод, 4-5 ролей, ~50 ТК в месяц.

## Решение

Используем SQLite (WAL mode) вместо PostgreSQL.

## Обоснование

| Фактор | SQLite | PostgreSQL |
|--------|--------|------------|
| Развёртывание | Один файл `bit_technolog.db` | Нужен сервер, systemd, бэкап |
| On-premise (ГОЗ) | Просто — файл рядом с app | Нужно настроить, права, доступ |
| Backup | `cp` | pg_dump, плагин |
| Производительность (≤50 RPS) | Достаточно | Избыточно |
| Размер БД | <100 MB | Любой |
| Operational cost | Ноль | Нужен DBA |

## Последствия

**Плюсы:**
- Ноль операционных затрат
- Простой backup (`cp bit_technolog.db backup.db`)
- Работает на любой Linux-машине
- Не требует отдельного сервиса в systemd

**Минусы:**
- Нет репликации (но не нужна — 1 завод)
- Нет параллельных writer'ов (WAL mode смягчает)
- При росте >100 RPS нужно переходить на Postgres

## Когда пересматривать

- Если пилот покажет >50 одновременных пользователей
- Если несколько заводов начнут использовать продукт
- Если понадобится репликация
