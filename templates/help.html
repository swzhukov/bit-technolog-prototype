📌 Как начать работу

  1. Переключитесь в роль "Технолог" через селектор в header (справа) или 3 кнопки на главной.
  2. Нажмите "＋ Деталь" в header → заполните форму:
    • Обозначение (обязательно) — например, АЦ-6,0-40
    • Наименование — что за деталь
    • Модель (шасси) — например, АЦ-6,0-40 на КАМАЗ-43118
    • Материал — например, Сталь 09Г2С
    • Размер, мм и Масса, кг — числа
    • Поверхностная обработка — например, Грунт ГФ-021
  3. Нажмите 💾 Сохранить → откроется карточка детали.
  4. В карточке: 🤖 AI-помощник (открыт по умолчанию) → ⚡ Draft → дождитесь 5-30 сек → операции появятся.
  5. Проверьте, отредактируйте, утвердите.
💡 Совет: используйте 3-step UX для лучшего качества: 🤔 Уточнить (AI задаст 3-5 вопросов) → ответьте → ⚡ Draft → ✨ Полная ТК.
{# ========== РОЛИ ========== #}

🎭 Роли — что видит каждая

Всего 7 ролей. Каждая видит свой набор кнопок и страниц. В header — цветной badge с вашей ролью, перепутать невозможно.

👨‍🔧 Технолог

Главный пользователь. Создаёт ТК.

Может: создавать детали, генерировать AI, редактировать, утверждать, импортировать/экспортировать.

Не может: админ-панель.

👑 Гл. технолог

Утверждающий. Подписывает ТК.

Может: всё, что технолог + утверждать как гл. технолог + возвращать в работу.

Не может: админ-панель.

🛡 Админ

Управляет системой.

Может: всё + /admin (8 страниц: пользователи, настройки, LLM-логи, ошибки, бэкапы).

📏 Нормировщик

Только просмотр. Проверяет экономику и нормы.

Может: смотреть детали, экономику, бенчмарки, отчёты.

Не может: создавать, генерировать, редактировать, утверждать.

🏭 Нач. цеха

Контролирует реализуемость.

Может: смотреть, утверждать как начальник цеха.

📐 Конструктор

Создаёт детали. Но не ТК.

Может: создавать детали, просматривать ТК когда технолог их сделает.

Не может: генерировать AI (кнопок нет).

🔍 ОТК (контролёр)

Контроль качества. Только просмотр.

Может: смотреть детали, контрольные точки, бенчмарки.

{# ========== WORKFLOW ТЕХНОЛОГА ========== #}

🛠 Workflow технолога (30 мин на деталь)

  1. Создать деталь — ＋ Деталь в header (5 мин)
  2. AI-помощник → 🤔 Уточнить — AI задаст 3-5 вопросов (10 сек) → ответьте (2 мин)
  3. ⚡ Draft — короткий маршрут, 3 операции, ~1₽ (10 сек + 1 мин проверка)
  4. ✨ Полная ТК — полный маршрут с учётом ответов, ~3₽ (30-60 сек + 5 мин проверка)
  5. Редактировать операции — кликните на название/оборудование/время (5 мин)
  6. Проверить экономику — вкладка 💰 Экономика: ставка, материал, накладные (2 мин)
  7. Утвердить — галки в чеклисте (4 пункта) → ✅ Утвердить (1 мин)
  8. Экспортировать — 📤 Загрузить (PDF), 📊 Excel, 📄 PDF, или /api/export/onec-csv (1 мин)
⚠ Пре-аппрув чеклист (4 пункта, обязательно для утверждения):
  1. Все операции заполнены (оборудование, время)
  2. Нет красных замечаний (все < 50% уверенности)
  3. Экономика проверена (ставка, материал, накладные)
  4. Связанные документы (чертежи, РС) в порядке
{# ========== ГЛ.ТЕХНОЛОГ ========== #}

👑 Workflow гл.технолога (утверждение)

  1. Переключитесь в роль Гл. технолог
  2. Главная → фильтр ?status=draft (проекты, ждут вашего утверждения)
  3. Откройте деталь → проверьте маршрут, экономику, замечания
  4. Если сомнения → Вернуть в работу (с комментарием)
  5. Если всё хорошо → Утвердить как гл. технолог
  6. ТК уходит в архив, редактирование ограничено
  7. Экспортируйте в PDF (с QR) для архива
{# ========== ДРУГИЕ РОЛИ ========== #}

📏 Что делает нормировщик

Только смотрит. Открывает детали → вкладка 💰 Экономика → сравнивает с бенчмарками. Если расхождение > 20% — пишет технологу. Кнопок не нажимает.

🏭 Что делает нач. цеха

Проверяет, что ТК реализуема в цеху (оборудование есть, время реалистично, материалы доступны). Утверждает как начальник цеха (после гл.технолога).

📐 Что делает конструктор

Создаёт детали через ＋ Деталь по чертежам. Сообщает технологу "Деталь X готова". Сам ТК не делает.

🔍 Что делает ОТК

Проверяет контрольные точки в операциях (поле control_points). Только просмотр. Никаких кнопок.

🛡 Что делает админ

Управляет системой. Доступ к /admin (8 страниц):

  • Пользователи — будущее (LDAP/AD)
  • Лог входов — кто заходил
  • LLM-вызовы — все вызовы AI, цена, ошибки
  • Настройки — YandexGPT API key, Telegram, SMTP
  • Ошибки — последние 50 ошибок сервера
  • Система — состояние (БД, RAG, диск)
  • Бэкап — ручной бэкап БД

{# ========== FAQ ========== #}

❓ Частые вопросы

Переключаю роль, но badge показывает старую

Обновите страницу (Ctrl+Shift+R — hard reload). Если не помогло — DevTools → Application → Cookies → удалите bit_role, повторите.

AI выдаёт мусор / не те операции

1) Проверьте входные данные (обозначение, материал, размер — чем точнее, тем лучше). 2) Используйте 3-step UX (Уточнить → Draft → Полная ТК). 3) Добавьте правила в "Правила технолога" в карточке детали. 4) Сообщите куратору — добавим в few-shot.

429 — дневной лимит LLM исчерпан

Лимит 200₽/день. Подождите до 00:00 МСК или попросите админа увеличить в настройках.

AI-блок не виден / кнопки не нажимаются

Проверьте роль в badge (справа вверху). Нормировщик, ОТК, нач. цеха, конструктор НЕ видят AI-блок — это by design (только создатель ТК).

Как импортировать детали из Excel?

Через API: POST /api/import/tk с JSON или файлом. См. API reference.

Как экспортировать в 1С?

В карточке детали → 📤 Загрузить → CSV для 1С (UTF-8, открыть в Excel через "Данные → Из текста").

Можно редактировать утверждённую ТК?

Нет (защита от случайных изменений). Чтобы отредактировать — Вернуть в работу → отредактировать → утвердить заново.

Что такое pre-approve checklist?

4 галочки, которые не дают утвердить ТК, пока не проверено всё: операции заполнены, нет красных замечаний, экономика проверена, документы в порядке. Это защита от утверждения "сырой" ТК.

Где искать логи / ошибки?

Админ → /admin/errors — последние 50 ошибок. Или SSH: sudo journalctl -u bit-technolog -n 100.

{# ========== TROUBLESHOOTING ========== #}

🆘 Что делать, если сломалось

  1. Откройте /health — покажет статус системы, версию, uptime, БД, LLM, Telegram, SMTP.
  2. Если status != "ok" — посмотрите /admin/errors (если есть права админа).
  3. Если кнопки не работают — откройте DevTools (F12) → Console, ищите красные ошибки.
  4. Полная диагностика — см. docs/18-troubleshooting.md.
  5. Связь с поддержкой — через куратора (Сергей) или issue в GitHub.
